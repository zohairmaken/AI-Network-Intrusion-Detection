# AI-Powered Network Intrusion Detection System using Machine Learning
---

## 1. Abstract
With the rapid expansion of network infrastructures, cyber attacks have become increasingly sophisticated. Traditional signature-based Intrusion Detection Systems (IDS) are no longer sufficient against zero-day attacks. This project presents an AI-Powered Network Intrusion Detection System (NIDS) that leverages Machine Learning to classify network traffic into five categories: Normal, DoS Attack, Port Scan, Brute Force, and Suspicious Activity. The system features a real-time Scapy packet sniffer, a custom feature extraction engine, and an interactive Streamlit dashboard designed to mimic a Security Operations Center (SOC).

## 2. Introduction
Network security is a critical concern in modern IT ecosystems. As attackers develop new methods to breach networks, the defense mechanisms must evolve. Artificial Intelligence, specifically Machine Learning (ML), provides the ability to detect anomalous behavior by learning from massive amounts of network traffic data. This project aims to build a robust, production-ready NIDS that not only detects threats but also provides severity scoring, automated logging, and a professional user interface for monitoring.

## 3. Literature Review
Previous research in NIDS heavily relied on datasets like KDD Cup 99 and NSL-KDD. Recent studies emphasize the use of modern datasets like CICIDS2017, which reflect current attack vectors. Researchers have successfully applied Support Vector Machines, Artificial Neural Networks, and Random Forests to these datasets. However, many proposed systems remain theoretical or lack real-time packet capturing and user-friendly visualization. This project bridges that gap by integrating ML models into a fully functional, real-time application.

## 4. Problem Statement
Existing open-source IDS solutions (like Snort or Suricata) are highly dependent on manually updated rule signatures. Conversely, many ML-based IDS projects are purely academic scripts that analyze static CSV files without the capability to monitor live network interfaces. There is a need for a unified, open-source solution that combines the predictive power of Machine Learning with real-time network sniffing and an intuitive graphical interface.

## 5. Objectives
- Develop a real-time packet capture engine using Python and Scapy.
- Extract statistical flow features (e.g., inter-arrival time, flow rate) from raw packets.
- Train multiple ML classifiers (Random Forest, Decision Tree, XGBoost) and compare their performance.
- Build a responsive, dark-themed Streamlit dashboard for live threat visualization.
- Implement a threat analyzer to calculate severity scores and recommend actions.

## 6. Methodology
The project follows a standard MLOps pipeline integrated with a continuous monitoring loop:
1. **Data Preprocessing:** Cleaning datasets, handling missing values, and encoding labels.
2. **Model Training:** Fitting ensemble tree-based models and saving them via `joblib`.
3. **Real-time Sniffing:** Operating a daemon thread to capture packets via a BPF filter.
4. **Feature Extraction:** Converting packet flags, sizes, and timestamps into a numerical vector matching the training data.
5. **Prediction & Alerting:** Passing the vector through the model and generating UI alerts.

## 7. Dataset Explanation
The system supports the CICIDS2017 and NSL-KDD datasets. For demonstration and rapid deployment, a highly realistic **Synthetic Dataset** generation module is included. It creates 10,000 samples across five classes using class-conditional statistical distributions (e.g., DoS traffic is generated with high packet rates and low inter-arrival times).

## 8. Algorithms Used
- **Random Forest:** Chosen for its robustness against overfitting and high accuracy on tabular network data.
- **Decision Tree:** Provided as a highly interpretable baseline model.
- **XGBoost:** Included for its superior execution speed and gradient boosting optimizations.

## 9. System Architecture
The software is highly modular:
- `core/`: Contains the sniffer, feature extractor, ML predictor, and threat analyzer.
- `ml/`: Handles the dataset preprocessing, training, and evaluation scripts.
- `app/`: Contains the Streamlit dashboard UI, authentication, and alert management.
- `config/`: Centralizes all constants, paths, and application settings.

## 10. Results
The models achieved the following metrics on the test data:
- **Random Forest:** Accuracy: 97.2%, Precision: 97.0%, Recall: 97.2%, F1-Score: 97.1%
- **Decision Tree:** Accuracy: 93.4%, Precision: 93.1%, Recall: 93.4%, F1-Score: 93.2%
- **XGBoost:** Accuracy: 96.5%, Precision: 96.3%, Recall: 96.5%, F1-Score: 96.4%

Confusion matrices and ROC curves generated by the system visually confirm the models' ability to distinguish between Normal traffic and various attack types with minimal false positives.
